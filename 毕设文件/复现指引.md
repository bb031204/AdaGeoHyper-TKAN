# HyperGKAN 论文复现建议（Reproduction Guide）

本文档旨在为 **《Hypergraph Kolmogorov–Arnold Networks for Station-Level Meteorological Forecasting》** 提供一份**工程化、可落地的复现指南**，帮助你从论文快速过渡到可运行代码与可验证实验结果。

---

## 1. 复现总体目标

复现目标建议分为三个层级：

### Level 1：结果复现（最低门槛）
- 能在 WeatherBench 子集上跑通 HyperGKAN
- MAE / RMSE 与论文 **数值量级一致**
- 不要求完全一致（±5% 可接受）

### Level 2：结构复现（推荐）
- 复现 **邻域超图 + 语义超图**
- 复现 **KAN-based Hypergraph Convolution**
- 复现 **Seq2Seq + GRU**

### Level 3：可扩展复现（进阶）
- 支持不同 K 值
- 支持 w/o-KAN / w/o-Sem 等消融
- 支持变量级别对比（Temp / Wind / Cloud）

## 2. 数据复现建议

### 2.1 数据集选择

论文使用：
- **WeatherBench 子集**
- 时间范围：2010-01-01 ~ 2018-12-31
- 时间分辨率：1 小时
- 变量：
  - Temperature
  - Cloud cover
  - Humidity
  - Wind speed

📌 **建议**：
- 若完整 WeatherBench 过大，可先抽取 **200–500 个 station**
- 保持 **station-level（非 grid-level）**

## 3. 超图构建复现要点（最容易踩坑）

### 3.1 邻域超图（Geographical Hypergraph）

**关键点：**
- 使用 **球面距离（Haversine / geodesic）**
- KNN 构建 hyperedge（论文中 K≈5 最优）

```python
from sklearn.neighbors import NearestNeighbors
```

注意：
- hyperedge ≠ edge
- 一个 hyperedge = K 个节点集合

---

### 3.2 语义超图（Semantic Hypergraph）

**核心思想**：
> “地理位置远，但时间序列相似”

实现建议：

```python
# flatten temporal dimension
feat = X_hist.reshape(N, -1)
dist = euclidean_distance(feat)
topk = KNN(dist)
```

📌 **建议**：
- 标准化（z-score）后再算距离
- 可缓存语义超图，避免每个 epoch 重算

---

## 4. KAN 复现建议（核心创新点）

### 4.1 不建议从零实现 KAN

KAN 非常容易数值不稳定，强烈建议：

- 使用官方或社区实现
- 推荐仓库：
  - https://github.com/KindXiaoming/pykan

---

### 4.2 KAN 使用方式（论文风格）

论文中：
- **KAN 替代 MLP / Linear**
- 用在：
  - Hypergraph Convolution
  - GRU 内部线性层

建议最小复现路径：

```python
KANLinear(in_dim, out_dim, grid=5, spline_order=3)
```

⚠️ 注意：
- 参数量 ≈ MLP 的 8 倍
- 网络要 **浅**

## 5. HyperGKAN Layer 复现结构

### 5.1 推荐模块拆分

```text
models/
├── kan.py
├── hypergraph.py
├── hypergkan_layer.py
├── hypergkan_gru.py

```

---

### 5.2 HyperGKAN Layer 伪代码

```python
Xg = KAN( Dv^-1/2 H W De^-1 H^T Dv^-1/2 X )
Xs = KAN( Dv~^-1/2 Hs Ws De~^-1 Hs^T Dv~^-1/2 X )

X = Linear(concat(Xg, Xs))
```

📌 **调试建议**：
- 先关掉 semantic hypergraph
- 再加 semantic
- 最后换 MLP → KAN

---

### 6.2 评价指标

```python
MAE = mean(|y - y_hat|)
RMSE = sqrt(mean((y - y_hat)^2))
```

建议：
- **逐变量评估**
- 不要只看 aggregate loss

## 7. 消融实验复现顺序（推荐）

1. Full HyperGKAN
2. w/o-KAN → MLP
3. w/o-Sem → only neighborhood
4. w/o-Nei → only semantic
5. K = {2,3,4,5,6}

---

## 8. 常见失败原因（真实踩坑总结）

- ❌ 把 hypergraph 当普通 graph
- ❌ KAN 层过深导致发散
- ❌ semantic hypergraph 每个 batch 重算
- ❌ 未区分 station-level vs grid-level
- ❌ MAE/RMSE 在 batch 维度算错

---

## 9. 复现成功的最低判断标准

你可以认为“复现成功”，如果：

- 云量 / 风速 **优于 STGCN / DCRNN**
- K≈5 最优
- w/o-KAN 明显退化
- 训练过程稳定，无 NAN
